# ResumeLoop — TL;DR

Local-first job-hunt tool: paste a JD, get a tailored 1-page DOCX resume + outreach drafts in under a minute — generated by **your own AI CLI**, on your machine, no keys, no cloud.

> Local-first per [ADR 0001](docs/adr/0001-pivot-to-local-first.md). Cloud v1 frozen on `legacy/cloud-v1` ([`DEPRECATED.md`](DEPRECATED.md)).

## The three seams

```
BRAIN          your AI CLI, headless — no API key
  lib/providers/  claude -p · codex exec · gemini -p · opencode run · http(ollama)
    └─ adapter: prompt → fenced JSON → Zod validate → retry once
         (claude.ts adds native --output-format json fast-path)

STATE          files canonical · DB is a rebuildable cache
  data/profile.json        resume content (single source of truth)
  data/jobs/*.md           JD + frontmatter (Action 0–6, visa, apply_url)
  data/evaluations/*.md    fit %, score, outreach brief
  data/resumes/            generated .docx / .pdf
    └─ .cache/index.db     SQLite index — delete it, rescan rebuilds it

ENGINE         generic + private-data split
  pipeline/buildv2.js              docx npm → ATS .docx (T()/TL() char gates)
  Playwright HTML→PDF template     pretty .pdf (no LibreOffice)
  validate: verb-uniqueness · em-dash ban · 1-page budget · fit gate
```

## System layers

```
Profile onboarding
  phase 1 (deterministic)  detect CLIs → pick provider → validate spawn → capture name/targets
  phase 2 (AI, once brain live)  paste CV / LinkedIn URL / GitHub handle
    └─ lib/ingest/  (extract → merge → conflict-resolve) → data/profile.json

Job intake
  Obsidian Web Clipper / Paste JD / drop .md  → data/jobs/*.md
    └─ fit scoring + role-track detection (genai / systems / fullstack / sre / IT-track)

Resume generation
  lib/providers/spine.ts  decideForJob()  → SpineDecision (Zod-validated)
    └─ buildv2.js          renderDocxBuffer() → .docx
         └─ Playwright      → .pdf
              └─ data/resumes/

Interfaces (share Zod schemas + types + index)
  Web (Next.js, 127.0.0.1)   /jobs · /chat · /profile · /settings
  TUI (Ink, optional)         onboarding Q&A · read-only pipeline dashboard · status bumps
```

## Key data

| File | Role |
|---|---|
| `data/profile.json` | All bullets, projects, work experience, skills (runtime source of truth) |
| `pipeline/master_resume_data.json` | Engine data **shape** / bootstrap template (not a real profile) |
| `pipeline/buildv2.js` | DOCX engine — takes selections, resolves metadata, enforces char limits |
| `lib/providers/spine.ts` | `SpineDecision` schema + JD→decision→DOCX round-trip |
| `CONTEXT.md` | Shared vocabulary + invariants (read before any task) |

## Trust & distribution

| Aspect | Local-first |
|---|---|
| Brain | Your AI CLI / local ollama — no stored key, no per-token cost |
| Auth | None — `127.0.0.1` + OS account is the trust boundary |
| Storage | Plain files in `data/` (git-trackable) + disposable SQLite index |
| PDF | Playwright headless Chromium (~150 MB, auto-installed) — no LibreOffice |
| Distribution | `npm i -g resumeloop` → `resumeloop init ~/career` → `resumeloop` (in progress) |
| Demo | `resumeloop.me` — anonymous, ephemeral, small self-hosted model via the same adapter |
